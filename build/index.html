IBM Maximo Application Suite Hands-on Lab

https://ibm.biz/techxchangelab2881

Authors: Arif Ali, Anantharaman Subramanian & Miryam Kofman - IBM Partner Technical Specialists North America.

This lab was first released in IBM's inaugural TechXchange event, held September 2023.

What is covered in this lab?‚Äã

When it comes to provisioning OpenShift in various environments, there exist multiple deployment patterns. For this lab, we have selected two patterns for OpenShift and two patterns for Maximo Application Suite: OpenShift automated and OpenShift interactive with MAS Ansible and MAS Pipeline. The Maximo Application Suite (MAS) will be provisioned on an on-premise OpenShift running on VMware in IBM's Tech Zone platform.

Platforms covered in this lab‚Äã

This lab covers seven distinct platforms. Click on the tabs to reveal the platforms covered in this lab.

  • IBM TechZone
  • IBM Deployment Automation Framework
  • Red Hat Assisted Installer for OpenShift
  • Red Hat Ansible Automation Platform
  • MAS Devops Ansible Collection
  • VMware vSphere for on-premise OpenShift
  • IBM Maximo Application Suite
IBM Technology Zone (TechZone) functions as the hub for IBM's go-to-market teams and the IBM Business Partner ecosystem, allowing them to provision and customize live demo environments on real-time infrastructure. In your role, you have been granted access to the live infrastructure of VMware vSphere through TechZone.
The tool provided by IBM for automating OpenShift deployment is a set of deployment scripts. These scripts facilitate the provisioning of OpenShift and Cloud Paks on different platforms. This automation framework is popularly referred to as "daffy," which stands for Deployment Automation Framework For You. In your role, you will utilize daffy to automate the provisioning of OpenShift using IPI (Installer-Provisioned Infrastructure) on VMware vSphere.
OpenShift Assisted Installer serves as a user-friendly solution for installing OpenShift on various platforms. In your role, you will begin by creating an account (or logging in) at https://console.redhat.com to obtain the software pull secret token. This token grants you the entitlement to build OpenShift and will be utilized for the OCP Automated pattern. Subsequently, you will utilize Red Hat's Assisted Installer to interactively provision OpenShift, leveraging the same token.
Red Hat Ansible Automation Platform serves as a comprehensive automation solution for configuring systems and deploying software. In your role, you will become acquainted with Ansible Roles and Playbooks in order to deploy the Maximo Application Suite.
MasDevOps is an Ansible collection designed to facilitate the automated deployment of the IBM Maximo Application Suite. In your role, a set of properly prepared Ansible scripts has been provided for you. These scripts enable you to deploy the Maximo Application Suite using Ansible.
VMware's vSphere platform is a virtualization technology (hypervisor). In your role, you will generate a group of virtual machines on VMware's vSphere platform, which is hosted by IBM's TechZone. These virtual machines will form a cluster consisting of both master and worker nodes. The Red Hat Assisted Installer will be employed to build the OpenShift cluster.
IBM Maximo Application Suite is a collection of applications designed for enterprise asset monitoring, management, predictive maintenance, and reliability planning. In your role, you will utilize Red Hat's Ansible automation to construct the Maximo Application Suite. This process entails provisioning the IBM Maximo Application Suite on one of the OpenShift clusters that you create, using Ansible automation.

Stage 2 of 2

The upcoming steps will guide you through the process of configuring Red Hat's podman utility for provisioning IBM's Maximo Application Suite (MAS) using Red Hat's Ansible DevOps code (MasDevOps). However, before you begin building MAS, it's crucial to confirm that your OpenShift installation has been successful through IBM's automation.

OpenShift log in‚Äã

  1. Return to the VS Code terminal and copy the output provided by the daffy automation at the conclusion of the successful installation during Stage 1 of 2 earlier. Paste this content into a Notepad file and save it, as you will need to refer to this information later.
Screenshot

  1. Copy your Web Console URL.
Screenshot

  1. Paste the URL into your Firefox browser and proceed. If you encounter any self-signed certificate warnings, accept them and continue. This will direct you to the OpenShift dashboard.
Screenshot

  1. Select Daffy htpasswd Provider. Enter ocpadmin as your username and copy/paste the password from the Notepad file. Click the Log in button to access OpenShift.
Screenshot

  1. Click on the three-dot menu and hide the banner from the view.
Screenshot

  1. The OpenShift dashboard might seem overwhelming initially. Make sure that your green check marks align with the screenshot. If they don't match, please raise your hand for assistance.

Ansible (One Click MAS)‚Äã

  1. Go back to the VS Code editor and create a new remote terminal.
Screenshot

  1. In the newly created terminal, which is running under admin user, change directory to mas8.
cd mas8
  1. Upload your AppPoint license to the mas8 directory (delete and replace).
Screenshot

  1. Run this command to fetch the author's convenient bash scripts for installing MAS using the MasDevOps One Click Ansible code.
bash <(curl -sL https://ibm.biz/BdyWKj)
  1. Run this command to launch Red Hat's podman utility for MasDevOps CLI. 
podman run -it --pull always -v ${PWD}:/scripts:Z --name ibmmas quay.io/ibmmas/cli:latest
note

This podman command will require about a minute or two to execute. It will then take you inside the container, and your bash prompt will change accordingly.

Screenshot

  1. Change directory.
cd /scripts
  1. From your notepad file saved earlier, copy/paste your specific OC Login command command.
Screenshot

  1. Install MAS Core via Ansible.
./1_core.sh
Screenshot

info

Upon running '1_core.sh', your terminal will initiate the Ansible process, which will proceed to deploy MAS Core automatically. The anticipated duration for this process is around 40 minutes, and it should conclude successfully. Throughout this process, there are three types of messages you might encounter: waiting messages, error messages, and success messages. As long as your terminal displays continuous activity, everything is proceeding as expected. Messages in green, yellow, teal, and grey colors are all normal indicators. However, if you encounter any messages in red and your command output abruptly ends, please raise your hand for assistance.

  1. After approximately 45 minutes, the Ansible automation process will successfully conclude, and you'll return to the command prompt. Scroll up to find the MAS URL along with the initial username and password details.
Screenshot

  1. In case you missed it, the initial (superuser) credentials can be found in the Secrets section of the mas-{instanceId}-core project within OpenShift.
