IBM Maximo Application Suite using Red Hat OpenShift Pipelines​

The upcoming steps will guide you through the process of configuring Red Hat's podman utility for provisioning IBM's Maximo Application Suite (MAS) using OpenShift Pipelines (Interactive MAS CLI). However, before you begin building MAS, it's crucial to confirm that your OpenShift installation has been successful through IBM's automation.

caution

The pipeline method is meant to be followed only on an OpenShift built using the IBM automation tool.

OpenShift log in​

  1. Return to the Terminal and copy the output provided by the daffy automation at the conclusion of the successful installation. Paste this content into a Text Editor and save it, as you will need to refer to this information later.
Screenshot

  1. Copy your Web Console URL. Paste the URL into your Firefox browser and proceed. If you encounter any self-signed certificate warnings, accept them and continue. This will direct you to the OpenShift dashboard.
Screenshot

  1. Select Daffy htpasswd Provider. Enter ocpadmin as your username and copy/paste the password from the Text Editor file. Click the Log in button to access OpenShift.
Screenshot

  1. Click on the three-dot menu and hide the banner from the view.
Screenshot

  1. The OpenShift dashboard might seem overwhelming initially. Make sure that your green check marks align with the screenshot. If they don't match, please raise your hand for assistance.

IBM Maximo Application Suite using Red Hat Ansible​

The upcoming steps will guide you through the process of configuring Red Hat's podman utility for provisioning IBM's Maximo Application Suite (MAS) using Red Hat's Ansible DevOps code (MasDevOps). However, before you begin building MAS, it's crucial to confirm that your OpenShift installation has been successful through IBM's automation.

caution

The direct Ansible method is meant to be followed only on an OpenShift built using the Assisted Installer method.

OpenShift log in​

  1. Return to the VS Code terminal and copy the output provided by the daffy automation at the conclusion of the successful installation during Stage 1 of 2 earlier. Paste this content into a Notepad file and save it, as you will need to refer to this information later.

Ansible (One Click MAS)​

  1. Go back to the VS Code editor and create a new remote terminal.
Screenshot

  1. In the newly created terminal, which is running under admin user, change directory to mas8.
cd mas8
  1. Upload your AppPoint license to the mas8 directory (delete and replace).
Screenshot

  1. Run this command to fetch the author's convenient bash scripts for installing MAS using the MasDevOps One Click Ansible code.
bash <(curl -sL https://ibm.biz/BdyWKj)
  1. Run this command to launch Red Hat's podman utility for MasDevOps CLI. 
podman run -it --pull always -v ${PWD}:/scripts:Z --name ibmmas quay.io/ibmmas/cli:7.1.0
note

This podman command will require about a minute or two to execute. It will then take you inside the container, and your bash prompt will change accordingly.

Screenshot

  1. Change directory.
cd /scripts
  1. From your notepad file saved earlier, copy/paste your specific OC Login command command.
Screenshot

  1. Install MAS Core via Ansible.
./1_core.sh
Screenshot

info

Upon running '1_core.sh', your terminal will initiate the Ansible process, which will proceed to deploy MAS Core automatically. The anticipated duration for this process is around 40 minutes, and it should conclude successfully. Throughout this process, there are three types of messages you might encounter: waiting messages, error messages, and success messages. As long as your terminal displays continuous activity, everything is proceeding as expected. Messages in green, yellow, teal, and grey colors are all normal indicators. However, if you encounter any messages in red and your command output abruptly ends, please raise your hand for assistance.

  1. After approximately 45 minutes, the Ansible automation process will successfully conclude, and you'll return to the command prompt. Scroll up to find the MAS URL along with the initial username and password details.
Screenshot

  1. In case you missed it, the initial (superuser) credentials can be found in the Secrets section of the mas-{instanceId}-core project within OpenShift.
