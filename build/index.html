Preparation (pre-event)​

daffy setup​

The following should be performed only once on a brand new Gym instance. This will prepare the daffy environment for the automated OCP installation.

Change user to root.

sudo su -

Copy and paste the block.

cat <<\EOF > pre-daffy.sh
#!/bin/bash
yum -y update
yum -y upgrade
yum -y install net-tools bind-utils curl nano vim tree wget unzip jq expect httpd-tools git grepcidr dos2unix openssl libaio binutils tar
cp /data/daffy/env/vmware-ipi-env.sh /data/daffy/env/vmware-ipi-env.sh.bak
sed -i '16i VSPHERE_NETWORKING_MACHINE_NETWORK_CIDR="192.168.252.0/24"' /data/daffy/env/vmware-ipi-env.sh
sed -i '24 {s/4.8.37/4.12.25/}' /data/daffy/env/vmware-ipi-env.sh
sed -i '25 {s/^/#/}' /data/daffy/env/vmware-ipi-env.sh
EOF

Give execute permission.

chmod +x pre-daffy.sh

Execute.

./pre-daffy.sh

The #IBMTechXChange Lab

What is involved in this lab​

There are several patterns available for the provisioning of OpenShift anywhere. We have picked two for this lab.

Pattern # 1Pattern # 2
OpenShift AutomatedOpenShift Interactive
IBM ToolRed Hat Tool
aka daffyaka Assisted Installer

Perspectives covered in this lab​

There are five different perspective covered in this lab.

  1. Explanation/Context
  2. Red Hat Cloud
  3. IBM Automation
  4. VMware vSphere
  5. Maximo Application Suite

1. Explanation/Context​

Instructor will provide the explanation and the context for this lab.

2. Red Hat Cloud​

You will be creating a Red Hat account if you don't already have one. The Red Hat account is needed to acquire the software pull secret token (entitlement). This token will be used for the OCP Automated pattern. Later, you will be using Red Hat's Assisted Installer to interactively provision OpenShift.

3. IBM Automation​

You will be using IBM's automation scripts to provision OpenShift on VMware.

4. VMware vSphere​

You will be building virtual machines on VMware's vSphere platform. These virtual machines will form a cluster of master and worker nodes. Red Hat's Assisted Installer will be used to build OpenShift cluster.

5. IBM Maximo Application Suite​

You will be using Ansible automation to provision IBM Maximo Application Suite on one of the OpenShift cluster you build.
