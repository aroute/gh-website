IBM TechXchange Hands-on Lab

https://ibm.biz/techxchangelab2881

What's included in this lab?​

When it comes to provisioning OpenShift in various environments, there exist multiple deployment patterns. For this lab, we have selected two: OpenShift automated and OpenShift interactive. The Maximo Application Suite will be provisioned using Ansible automation.

Pattern # 1Pattern # 2Pattern # 3
OpenShift AutomatedOpenShift InteractiveMaximo Application Suite
IBM ToolRed Hat ToolRed Hat Ansible Tool
aka daffyaka Assisted Installeraka MasDevOps
Pattern 1 employs IBM's automation framework, commonly referred to as "daffy," to construct an OpenShift cluster. The installation of the Maximo Application Suite (MAS core) will follow this approach.Pattern 2 utilizes Red Hat's Assisted Installer technology to build OpenShift cluster.The deployment of the Maximo Application Suite (MAS) will be carried out using IBM's MasDevOps code, which is built upon Red Hat's Ansible automation framework.

Platforms covered in this lab​

There are six different platforms covered in this lab.

  • Platforms →
  • Jumpbox
  • IBM Automation
  • Red Hat Assisted Installer
  • Red Hat Ansible
  • VMware vSphere
  • IBM Maximo Application Suite
Click on the tabs to uncover the platforms covered in this lab.
The IBM Event Cloud's Jumpbox serves as a Windows 10 workstation for executing all the necessary tasks. This Jumpbox is a Windows 10 virtual machine from which all the commands are being carried out for this lab.
To access the assisted installer technology at https://console.redhat.com, you'll need to create a Red Hat account if you don't already have one. This account is essential for acquiring the software pull secret token, which provides you with the entitlement to build OpenShift. This token will be used for the OCP Automated pattern. Later on, you will employ Red Hat's Assisted Installer to interactively provision OpenShift, using the same token.
The tool provided by IBM for automating OpenShift deployment. These scripts enable you to provision OpenShift on VMware. This framework is commonly known as "daffy," which stands for Deployment Automation Framework For You.
IBM Tech Zone Gymnasium serves as a virtualization platform for running OpenShift. You will create a set of virtual machines on VMware's vSphere platform, which is hosted by IBM's Technology Zone (also referred to as Tech Zone Gym). These virtual machines will constitute a cluster composed of both master and worker nodes. The Red Hat Assisted Installer will be used to construct the OpenShift cluster.
Red Hat Ansible Automation Platform is an end-to-end automation platform to configure systems and deploy software.
For building the Maximo Application Suite, you will employ Red Hat's Ansible automation. This approach involves using Ansible automation to provision the IBM Maximo Application Suite on one of the OpenShift clusters that you build.

You're all set. Let's get started!

1. OpenShift Automated via IBM tool​

Stage 1 of 2

The subsequent steps will guide you through the process of installing OpenShift using IBM's automation framework scripts (daffy).

Log in​

  1. Connect to your WireGuard VPN.

  2. Locate your Gym's password.

  3. Open PowerShell and run.

type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh admin@192.168.252.2 "cat >> .ssh/authorized_keys"

  1. Type your Gym's password.

  2. Validate passwordless login. You should be able to log into the Linux environment (bastion node) without requiring a password.

ssh admin@192.168.252.2
  1. Type exit and close your PowerShell window.

Working Environment​

  1. Launch VS Code editor. Click to select Remote Explorer icon.
Screenshot

  1. Hover over "admin" under the bastion node, and then click the arrow to select Connect in Current Window...
Screenshot

  1. In less than a minute, your Linux environment (bastion node) will be available, with the file explorer on the left and a bash terminal on the bottom-right.
Screenshot

  1. On the remote bash terminal, switch user to root.
sudo su -

Scripted Installation​

  1. Update the daffy environment.
bash <(curl -sL https://ibm.biz/BdyWmv)
caution

Step #2 initiates the automated installation of OpenShift, which usually takes around 45 minutes. However, the script will prompt you for your pull secret token within a few minutes. Please read the following instructions carefully and proceed accordingly.

  1. Copy/paste this command on the VS Code's remote terminal. DO NOT hit Enter yet.
/data/daffy/ocp/build.sh vmware-ipi
  1. From the Windows jumpbox, right-click to open your earlier downloaded Pull Secret file in a Notepad. Copy all the content to the clipboard: shortcut Ctrl+a + Ctrl+c.
Screenshot

  1. In the VS Code's Terminal window, press Enter after pasting the command from step 1 mentioned above.
note

After a few minutes, the OCP automation script will prompt you to paste your pull secret. As per step #2 mentioned earlier, the content of your pull secret is in your clipboard.

  1. When the automation script prompts you to input your pull secret, paste the content from your clipboard and press Enter.
Screenshot

info

This phase of the Automated OpenShift installation setup is now finished. Allow the automation to continue running in the terminal. You will return to this process in approximately 45 minutes when the instance is prepared.

3. IBM Maximo Application Suite using Red Hat Ansible​

Stage 2 of 2

The upcoming steps will guide you through the process of configuring Red Hat's podman utility for provisioning IBM's Maximo Application Suite (MAS) using Red Hat's Ansible DevOps code (MasDevOps). However, before you begin building MAS, it's crucial to confirm that your OpenShift installation has been successful through IBM's automation.

OpenShift log in​

  1. Return to the VS Code terminal and copy the output provided by the daffy automation at the conclusion of the successful installation. Paste this content into a Notepad file and save it, as you will need to refer to this information later.
Screenshot

  1. Copy your Web Console URL.
Screenshot

  1. Paste the URL into your Firefox browser and continue. Accept the Security Warning by clicking on "Advanced" and then "Accept and continue." Repeat this process once more when prompted. This will lead you to the OpenShift dashboard.
Screenshot

  1. Select Daffy htpasswd Provider. Enter ocpadmin as your username and copy/paste the password from the Notepad file. Click the Log in button to access OpenShift.
Screenshot

  1. Click on the three-dot menu and hide the banner from the view.
Screenshot

  1. The OpenShift dashboard might seem overwhelming initially. Make sure that your green check marks align with the screenshot. If they don't match, please raise your hand for assistance.
Screenshot

Ansible​

  1. Go back to the VS Code editor and create a new remote terminal.
Screenshot

  1. In the newly created terminal, which is running under admin user, change directory to mas8.
cd mas8
Screenshot

  1. Execute this command to retrieve the author's bash scripts for installing MAS using the MasDevOps Ansible code.
bash <(curl -sL https://ibm.biz/BdyWKj)
  1. Run the command to launch Red Hat's podman utility.
note

The podman command will require about a minute or two to execute. It will then take you inside the container, and your bash prompt will change accordingly.

podman run -it --pull always -v ${PWD}:/scripts:Z --name mastechex quay.io/ibmmas/cli
Screenshot

  1. Change directory.
cd /scripts
  1. Log in to OpenShift via Command line terminal. Locate your oc login ... command from your notepad file saved earlier.
oc login ...
Screenshot

  1. Install MAS Core via Ansible.
./1_core.sh
Screenshot

info

Upon running '1_core.sh', your terminal will initiate the Ansible process, which will proceed to deploy MAS Core automatically. The anticipated duration for this process is around 40 minutes, and it should conclude successfully. Throughout this process, there are three types of messages you might encounter: waiting messages, error messages, and success messages. As long as your terminal displays continuous activity, everything is proceeding as expected. Messages in green, yellow, teal, and grey colors are all normal indicators. However, if you encounter any messages in red and your command output abruptly ends, please raise your hand for assistance.
