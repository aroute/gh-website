IBM TechXchange Hands-on Lab

What's included in this lab?​

When it comes to provisioning OpenShift in various environments, there exist multiple deployment patterns. For this lab, we have selected two: OpenShift automated and OpenShift interactive. The Maximo Application Suite will be provisioned using Ansible automation.

Pattern # 1Pattern # 2Pattern # 3
OpenShift AutomatedOpenShift InteractiveMaximo Application Suite
IBM ToolRed Hat ToolRed Hat Ansible Tool
aka daffyaka Assisted Installeraka MasDevOps
Pattern 1 employs IBM's automation framework, commonly referred to as "daffy," to construct an OpenShift cluster. The installation of the Maximo Application Suite (MAS core) will follow this approach.Pattern 2 utilizes Red Hat's Assisted Installer technology to build OpenShift cluster.The deployment of the Maximo Application Suite (MAS) will be carried out using IBM's MasDevOps code, which is built upon Red Hat's Ansible automation framework.

Platforms covered in this lab​

There are five different platforms covered in this lab.

  • Platforms ➡️
  • Jumpbox
  • Red Hat Cloud
  • IBM Automation
  • VMware vSphere
  • IBM Maximo Application Suite
Click to view details on these platforms.
IBM Event Cloud's Jumpbox: Windows 10 workstation to perform all the tasks. The Jumpbox is the Windows 10 virtual machine from which all the commands are being executed for this lab.
https://console.redhat.com for assisted installer technology: You will need to create a Red Hat account if you don't already have one. The Red Hat account is necessary to obtain the software pull secret token, which grants you entitlement to build OpenShift. This token will be utilized for the OCP Automated pattern. Subsequently, you will use Red Hat's Assisted Installer to provision OpenShift interactively, using the same token.
IBM tool to build OpenShift using automation: You will utilize IBM's deployment automation framework scripts to provision OpenShift on VMware. This framework is commonly referred to as "daffy" (Deployment Automation Framework For You).
A virtualization platform for running OpenShift: IBM Tech Zone Gymnasium as a virtualization platform for running OpenShift. You will construct a group of virtual machines on VMware's vSphere platform, which is hosted by IBM's Technology Zone (also known as Tech Zone Gym). These virtual machines will comprise a cluster of master and worker nodes. Red Hat's Assisted Installer will be employed to build the OpenShift cluster.
Red Hat's Ansible to build Maximo Application Suite: You will be using Ansible automation to provision the IBM Maximo Application Suite on one of the OpenShift clusters you build.

Pattern 1: OpenShift Automated via IBM tool​

Stage 1 of 2

The following steps will show you how to install OpenShift using IBM's automation framework scripts (daffy).

Passwordless log in (SSH keys)​

  1. Connect to your WireGuard VPN.

  2. Locate your Gym's password.

  3. Open PowerShell and run.

type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh admin@192.168.252.2 "cat >> .ssh/authorized_keys"

  1. Type your Gym's password.

  2. Validate passwordless log in. You should be able to log in to the Linux environment (bastion node) without the password.

ssh admin@192.168.252.2
  1. Type exit and close your PowerShell window.

VS Code Editor​

  1. Launch VS Code editor. Click to select Remote Explorer icon.
Screenshot

  1. Mouse-over to admin under bastion and click the arrow to select Connect in Current Window ...
Screenshot

  1. Within a minute, you will have your Linux (bastion node) environment with file explorer on the left, and bash terminal on the bottom-right.
Screenshot

  1. On the remote bash terminal, switch user to root.
sudo su -

Install OpenShift via automation (daffy script)​

  1. Update the daffy environment.
bash <(curl -sL https://ibm.biz/BdyWmv)
caution

The following step # 2 starts the automated installation of OpenShift which takes approximately 45 minutes. However, the script will ask you for your pull secret token within a few minutes. Carefully read the following and proceed forward.

  1. Copy/paste this command on the VS Code's remote terminal. DO NOT hit Enter yet.
/data/daffy/ocp/build.sh vmware-ipi
  1. From the Windows jumpbox, right-click to open your earlier downloaded Pull Secret file in a Notepad. Copy all the content to the clipboard: shortcut Ctrl+a + Ctrl+c.
Screenshot

  1. From the VS Code's Terminal window, hit Enter on the above pasted command from step 1.
note

Within a few minutes, the OCP automation script will prompt you to paste your pull secret. Based on step # 2 above, your pull secret content is in your clipboard.

  1. When the automation script asks you to provide your pull secret, simply paste the content of your clipboard and hit Enter.
Screenshot

info

This part of the Automated OpenShift installation setup is complete. Let the automation run on its own in the terminal. You will come back to this in 45 minutes when the instance is ready.

3. IBM Maximo Application Suite using Red Hat Ansible​

Stage 2 of 2

The following steps will show you how to set up Red Hat's podman utility to provision IBM's Maximo Application Suite (MAS) using Red Hat's Ansible devops code (MasDevOps). However, before you start building MAS, you need to ensure your OpenShift is installed successfully using IBM's automation.

daffy provided output​

  1. Go back to the VS Code terminal and copy the output (credentials and URLs) presented by the daffy automation at the end of the successful installation. Paste the content into a Notepad file. Save the notepad file as you will be referring to this information later.
Screenshot

OpenShift log in (Web Console)​

  1. Copy your Web Console URL.
Screenshot

  1. Paste the URL into your Firefox browser and proceed. Accept the Security Warning by clicking on Advanced and clicking Accept and continue. Repeat this once again when prompted. You will be presented with the OpenShift dashboard.
Screenshot

  1. Click on Daffy htpasswd Provider. Type ocpadmin as your username and copy/paste the password from the notepad file. Click Log in button to log in to OpenShift.
Screenshot

  1. Click on the three-dot menu and hide the banner from the view.
Screenshot

  1. OpenShift's dashboard may look very busy at first. Ensure your green check marks match the screenshot. Raise your hand if they do not.
Screenshot

Deployment scripts​

  1. Go back to the VS Code editor and create a new remote terminal.
Screenshot

  1. In the newly created terminal, which is running under admin user, change directory to mas8.
cd mas8
Screenshot

  1. Run this command to pull author's bash scripts to install MAS using masdevops Ansible code.
bash <(curl -sL https://ibm.biz/BdyWKj)

Podman utility for Ansible/MasDevOps code​

  1. Run the command to launch Red Hat's podman utility.
note

The podman command will take a minute or two to run. It will take you inside the container and your bash prompt will change.

podman run -it --pull always -v ${PWD}:/scripts:Z --name mastechex quay.io/ibmmas/cli
Screenshot

  1. Change directory.
cd /scripts

OpenShift log in (Command Terminal)​

  1. Log in to OpenShift via Command line terminal. Locate your oc login ... command from your notepad file saved earlier.
oc login ...
Screenshot

Start Ansible​

  1. install MAS Core via Ansible.
./1_core.sh
Screenshot

info

When you run '1_core.sh', your terminal will launch the Ansible process, which will automatically deploy MAS Core. We anticipate that the process will take ~40 minutes and will end successfully. There are three types of messages you should expect to see: waiting, errors, and successful. As long as your terminal is moving, everything is fine. Messages in the colors green, yellow, teal, and grey are all acceptable. If you see anything in red and your command output ends abruptly, raise your hand.
